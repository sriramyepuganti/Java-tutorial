IOC: inversion of control

Inversion of control is a pattern used to decouple the dependencies between layers and components in the system.

Introduction

  • Apache Maven is a software project management and comprehension tool
  • Based on the concept of a project object model (POM)
  • Maven can manage a project's build, reporting and documentation from a central piece of information.

  • project: This is the root element of the pom.xml file. It defines the basic information about the project, such as the project name, version, and description.
  • modelVersion: This specifies the version of the Project Object Model (POM) that the file uses. It should be set to 4.0.0 for Maven 2 and newer.
  • groupId: This is a unique identifier for the project group or organization. It typically follows the reverse domain name notation, such as com.example.
  • artifactId: This is a unique identifier for the project artifact (or module). It is usually the name of the project, such as my-project.
  • version: This is the version of the project artifact. It typically follows the Major.Minor.Patch format, such as 1.0.0.
  • dependencies: This is a list of dependencies that the project requires to build and run. Each dependency has its own groupId, artifactId, and version specified.
  • build: This element describes how to build the project. It can include information about the build process, such as the source directory, the output directory, and plugins to use.
  • plugins: This is a list of plugins that the project uses to build and run. Each plugin has its own groupId, artifactId, and version specified.
  • repositories: This is a list of repositories where Maven can find the project's dependencies and plugins. By default, Maven looks in the central repository, but additional repositories can be specified here.
  • profiles: This is a way to define different build settings for different environments (such as development, testing, or production). Profiles can include dependencies, plugins, and other build settings.
